# @openbag/skill-cli

Command-line tool for creating, validating, testing, and publishing OpenBag skills.

## Installation

```bash
npm install -g @openbag/skill-cli
# or run directly without installing:
npx @openbag/skill-cli <command>
```

## Commands

### `openbag skill init`

Interactive wizard that scaffolds a new skill directory.

```bash
openbag skill init
# Prompts for name, description, author, permissions, and category.
# Creates: SKILL.yaml, index.js, test/index.js, README.md
```

You can target a specific parent directory:

```bash
openbag skill init --dir ~/my-skills
```

### `openbag skill validate`

Validates a `SKILL.yaml` manifest. Uses the JSON schema at `sdk/skill-manifest.schema.json`
when available; falls back to structural checks otherwise.

```bash
# Validate SKILL.yaml in the current directory
openbag skill validate

# Validate a specific file
openbag skill validate --file path/to/SKILL.yaml

# Use a custom schema
openbag skill validate --schema path/to/schema.json
```

### `openbag skill test`

Discovers and runs all `.js` files inside the skill's `test/` directory using Node.js.

```bash
# Run tests for the skill in the current directory
openbag skill test

# Run tests for a skill in a different directory
openbag skill test --dir ~/my-skills/skill-route
```

### `openbag skill publish`

Guides you through publishing a skill to the **ClawHub-BR** registry.
Automated publishing via API token is on the roadmap; for now the command prints
step-by-step manual instructions.

```bash
openbag skill publish
openbag skill publish --file path/to/SKILL.yaml
```

### `openbag skill list`

Lists all skills currently installed under `~/.openbag/skills/`.

```bash
openbag skill list
```

Example output:

```
Installed skills (/home/entregador/.openbag/skills):

  skill-verify            core          v1.2.0    Responds to citizen verification requests
  skill-rep               core          v1.0.1    Civic reputation tier calculation
  skill-motoboy-map       community     v0.3.0    Real-time route sharing for motoboys
```

## SKILL.yaml structure

A minimal manifest generated by `openbag skill init`:

```yaml
apiVersion: openbag.org/v1alpha1
kind: Skill
metadata:
  name: skill-example
  version: 0.1.0
  description: An example skill
  author: contributor
  category: community
  license: MIT
spec:
  permissions:
    - ble-scan
    - location
  entrypoint: index.js
  hooks:
    onInstall: null
    onUninstall: null
    onVerify: null
```

### Available permissions

| Permission      | What it grants                              |
|-----------------|---------------------------------------------|
| `ble-scan`      | Scan for Bluetooth Low Energy devices       |
| `nfc-read`      | Read NFC tags                               |
| `nfc-write`     | Write NFC tags                              |
| `location`      | Access GPS / location data                  |
| `notifications` | Send local push notifications               |
| `network`       | Make outbound HTTP/HTTPS requests           |

## Development workflow

```bash
# 1. Scaffold
openbag skill init

# 2. Implement your logic in index.js

# 3. Validate the manifest
openbag skill validate

# 4. Run tests
openbag skill test

# 5. Publish
openbag skill publish
```
